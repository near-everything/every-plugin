The every-plugin core defines the plugin interface and type system that enables composable, type-safe plugin architectures.

## Plugin Interface

All plugins implement the `Plugin<TInput, TOutput, TConfig>` interface:

```typescript
interface Plugin<TInputSchema, TOutputSchema, TConfigSchema> {
  readonly id: string;
  readonly type: PluginType;
  readonly inputSchema: TInputSchema;
  readonly outputSchema: TOutputSchema;
  readonly configSchema: TConfigSchema;
  
  initialize(config?: z.infer<TConfigSchema>): Effect.Effect<void, ConfigurationError>;
  execute(input: z.infer<TInputSchema>): Effect.Effect<z.infer<TOutputSchema>, PluginExecutionError>;
  shutdown(): Effect.Effect<void, never>;
}
```

## Plugin Types

**Source**: Fetch data from external APIs with oRPC contracts and state management.

**Transformer**: Process and transform data between different formats.

**Distributor**: Send data to external systems or storage.

## Type System

Plugins use Zod schemas for runtime validation:

```typescript
// Input validation
const inputSchema = z.object({
  query: z.string().min(1),
  limit: z.number().optional()
});

// Output validation  
const outputSchema = z.object({
  items: z.array(ItemSchema),
  nextCursor: z.string().optional()
});

// Configuration validation
const configSchema = createConfigSchema(
  z.object({ timeout: z.number().optional() }), // variables
  z.object({ apiKey: z.string().min(1) })       // secrets
);
```

## Effect Integration

Plugins use Effect for composable async operations with automatic resource management:

```typescript
execute(input: Input): Effect.Effect<Output, PluginExecutionError> {
  return Effect.gen(function* () {
    const logger = yield* PluginLoggerTag;
    
    yield* logger.logInfo("Processing request", { input });
    
    const result = yield* Effect.tryPromise({
      try: () => this.client.process(input),
      catch: (error) => new PluginExecutionError(`Processing failed: ${error}`, true)
    });
    
    return { success: true, data: result };
  });
}
```
